# 🚀 SOLID Principles in C# (.NET)

A practical implementation of the **SOLID Principles** using C# and .NET.
This repository demonstrates how to write clean, maintainable, and scalable code by applying the five core object-oriented design principles.

---

## 📌 Overview

This project provides clear and simple examples of each **SOLID Principle**, showing how applying them improves code quality and maintainability.

The goal is to:

* Understand each principle with real code examples
* Learn how to refactor bad design into clean architecture
* Write scalable and testable applications

---

## 🧠 SOLID Principles Included

### 🧩 Single Responsibility Principle (SRP)

> A class should have only one reason to change.

* Each class handles one specific responsibility
* Improves readability and maintainability

---

### 🔓 Open/Closed Principle (OCP)

> Software entities should be open for extension but closed for modification.

* Add new features without modifying existing code
* Achieved using abstraction and polymorphism

---

### 🔄 Liskov Substitution Principle (LSP)

> Subtypes must be substitutable for their base types.

* Derived classes should not break base class behavior
* Ensures reliability in inheritance

---

### 🧱 Interface Segregation Principle (ISP)

> Clients should not be forced to depend on interfaces they do not use.

* Split large interfaces into smaller, specific ones
* Reduces unnecessary dependencies

---

### 🔁 Dependency Inversion Principle (DIP)

> High-level modules should not depend on low-level modules.

* Depend on abstractions instead of concrete implementations
* Promotes loose coupling and flexibility
